# 파인딩 상세

## 개요

개별 보안 발견사항의 상세 정보를 확인하고 조치 상태를 관리하는 페이지입니다.

## 화면 구성

### 상단: 브레드크럼

`← 뒤로 > 도메인 > 발견사항 제목` 형태의 경로가 표시됩니다. 도메인을 클릭하면 해당 도메인 상세로 이동합니다.

### 헤더 카드

* **제목**: 발견사항 이름
* **심각도 배지**: Critical(빨강), High(주황), Medium(노랑), Low(파랑)
* **상태 배지**: 미조치, 조치 중, 조치 완료 등
* **유형 배지**: CVE, 자격증명 유출, 포트 스캔 등
* **CVE ID**: NVD 링크 (해당하는 경우)
* **발견일**: 처음 탐지된 날짜와 시간

### 메인 콘텐츠 (왼쪽)

| 섹션 | 내용 |
|------|------|
| 설명 | 발견사항에 대한 상세 설명 |
| 개선 방안 | 권장 조치 방법 |
| 증거 | 기술적 증거 (소스, 날짜 등) |
| CPE 식별자 | 해당하는 소프트웨어 식별자 (CPE) |
| 참조 | NVD, CVE.org 등 외부 참조 링크 |

### 사이드바 (오른쪽)

| 섹션 | 내용 |
|------|------|
| CVSS 점수 | CVSS 점수 바 + 심각도 레이블 |
| CVSS 벡터 | 공격 벡터, 복잡도 등 8개 항목 |
| CWE 분류 | CWE ID + 이름 + Top 25 여부 |
| 악용 확률 | EPSS 추정 확률 (CVE만 해당) |
| 영향받는 구성요소 | 영향받는 시스템 구성요소 |
| 메타데이터 | ID, 유형, 자산 정보 |
| 상태 변경 | 상태 업데이트 버튼 |
| AI 분석 | AI 기반 분석 결과 (제공 시) |

## 상태 변경

오른쪽 사이드바의 **상태 변경** 섹션에서 발견사항의 처리 상태를 변경할 수 있습니다:

| 상태 | 설명 |
|------|------|
| 미조치 (Open) | 아직 대응하지 않은 상태 |
| 조치 중 (In Progress) | 현재 대응 진행 중 |
| 조치 완료 (Resolved) | 문제가 해결된 상태 |
| 무시 (Ignored) | 의도적으로 무시 (위험 수용) |
| 오탐 (False Positive) | 실제 위협이 아닌 것으로 판단 |

{% hint style="warning" %}
조치 완료, 무시, 오탐으로 변경 시 사유를 입력할 수 있습니다. 감사 추적을 위해 사유를 기록하는 것을 권장합니다.
{% endhint %}
