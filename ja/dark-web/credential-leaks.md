# 認証情報の流出

## 概要

認証情報の流出は、ダークウェブで最も頻繁に検出される脅威の種類です。組織メンバーのメールアドレスとパスワードがダークウェブのフォーラムやマーケットで取引されているケースを検出します。

## リスクレベル

| 種別 | リスクレベル | 説明 |
|------|--------|------|
| 平文パスワード | 🔴 Critical | パスワードが暗号化されずに露出。即時変更が必要 |
| ハッシュパスワード | 🟠 High | 復号化の可能性あり。パスワード変更を推奨 |
| メールアドレスのみ露出 | 🟡 Medium | フィッシング/ソーシャルエンジニアリングのリスク |

## 推奨対応措置

1. **即時**: 流出したアカウントのパスワードを強制変更
2. **短期**: 多要素認証 (MFA) を有効化
3. **長期**: パスワードポリシーの強化、流出モニタリングの継続

## 流出元の理解

SafeIntelligenceは、以下のソースから認証情報の流出を検出します：

* **BreachForum**: 大規模データ漏洩取引フォーラム
* **Telegramチャンネル**: 流出情報共有チャンネル
* **Phishing Kit Tracker**: フィッシングキットから収集された情報
* **その他のダークウェブソース**: 各種ダークウェブマーケットおよびフォーラム

## 注意事項

{% hint style="warning" %}
流出した認証情報は、他のサービスでも同様に使用されている可能性があります（Credential Stuffing）。流出が発見された場合は、該当メールアドレスで使用しているすべてのサービスのパスワードを変更することを推奨します。
{% endhint %}
