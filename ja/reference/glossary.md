# 用語集

## 一般用語

| 用語 | 説明 |
|------|------|
| **ASM** | Attack Surface Management。攻撃対象領域管理。外部に露出した資産を識別し、セキュリティリスクを管理する活動 |
| **OSINT** | Open Source Intelligence。公開情報を活用したインテリジェンス収集 |
| **CTI** | Cyber Threat Intelligence。サイバー脅威インテリジェンス |

## 資産関連

| 用語 | 説明 |
|------|------|
| **ドメイン** | 組織が所有するインターネットドメイン（例: example.com） |
| **サブドメイン** | ドメイン配下のホスト名（例: api.example.com） |
| **スコープ** | 監視対象として設定されたドメインの範囲 |
| **資産 (Asset)** | ドメイン、サブドメイン、IPなどの監視対象リソース |

## 脆弱性関連

| 用語 | 説明 |
|------|------|
| **CVE** | Common Vulnerabilities and Exposures。公開された既知のセキュリティ脆弱性識別子 |
| **CVSS** | Common Vulnerability Scoring System。脆弱性の深刻度スコア (0-10) |
| **CWE** | Common Weakness Enumeration。ソフトウェアセキュリティ上の弱点の分類体系 |
| **CPE** | Common Platform Enumeration。ソフトウェア/ハードウェア識別の標準規格 |

## 深刻度

| 等級 | CVSS範囲 | 説明 |
|------|----------|------|
| **Critical** | 9.0 - 10.0 | 即時対応が必要な致命的脆弱性 |
| **High** | 7.0 - 8.9 | 迅速な対応が必要な高リスク |
| **Medium** | 4.0 - 6.9 | 計画的な対応が必要な中程度のリスク |
| **Low** | 0.1 - 3.9 | 低リスク、監視レベル |

## ダークウェブ関連

| 用語 | 説明 |
|------|------|
| **ダークウェブ** | 一般のブラウザではアクセスできないインターネット領域 |
| **認証情報の流出** | メールアドレス＋パスワードがダークウェブ上で発見されたケース |
| **Credential Stuffing** | 流出した認証情報を他のサービスに対して使用する攻撃手法 |
| **感染デバイス** | マルウェアに感染し、情報が流出したデバイス |
| **フィッシングキット** | フィッシング攻撃に使用されるツールセット |

## セキュリティ等級

| 等級 | スコア範囲 | 意味 |
|------|----------|------|
| **A** | 75以上 | 優れたセキュリティ状態 |
| **B** | 60 - 74 | 良好なセキュリティ状態 |
| **C** | 40 - 59 | 改善が必要なセキュリティ状態 |
| **D** | 20 - 39 | 脆弱なセキュリティ状態 |
| **F** | 20未満 | 深刻に脆弱なセキュリティ状態 |
